GIT CHEATSHEET:

FOR EACH NEW FEATURE OR CHANGE:

  • Create a new branch
  • Make your changes in the code

PULL MASTER BRANCH TO FEATURE BRANCH:

(Be sure to have clean status before this step)

  • git checkout master
  • git pull origin master
  • git checkout branch-name
  • git merge master
  • git push origin branch-name

    • (**origin is the nickname of my GitHub)

CREATE PULL REQUEST:

  • In terminal: hub browse
  • Click on branches in your browser
  • Find your branch
  • Click create pull request
  • Assign to a reviewer
  • Create a meaningful message explaining what you did and how to test

REVIEW OF CODE:

  • The reviewer reviews the pull request
  • Read the meaningful message explaining how to test the code and what was added

TO TEST:

  • git fetch (downloads all branches)
  • git checkout pull-request-branch-name
  • The reviewer merges the code

AFTER MERGE IS DONE, YOU SHOULD:

    (These steps are for the person who submitted the pull request)

  • git checkout master
  • git pull origin master
  • git sweep (cleans of the merged branches to ensure you donâ€™t work on a finished branch, and to keep GitHub clean)
  • git checkout -b new-branch-name (you start working on the next feature)

AFTER A MERGE IS DONE, YOUR TEAM SHOULD:

    (These steps are for the team to update the branches they are working in to a new version of the master. This can be in current features they are working on too.)

  • git checkout master
  • git pull origin master
  • git checkout branch-name
  • git merge master (this merges the code FROM master into your branch-name)
  • git status
  • git push origin your-branch-name
